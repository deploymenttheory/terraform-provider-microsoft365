# microsoft365_graph_beta_identity_and_access_cross_tenant_access_partner_group_sync_settings (Resource)

Manages cross-tenant group synchronization settings for a specific partner tenant in Microsoft Entra ID using the `/policies/crossTenantAccessPolicy/partners/{id}/identitySynchronization` endpoint. This resource is used to configure whether groups from a partner Microsoft Entra tenant can be synchronized to your tenant.

## Important Notes

- **Partner-Specific Configuration**: This resource manages group synchronization settings for a specific partner tenant. Each partner tenant requires its own resource instance.
- **Tenant ID Validation**: The `tenant_id` is validated against Microsoft Entra ID to ensure it represents a valid organization before creating the resource.
- **RequiresReplace**: Changing the `tenant_id` will force replacement of the resource, as it's used as the identifier.
- **Group Sync Direction**: Currently, only inbound group synchronization (from partner to your tenant) is supported via the `group_sync_inbound` configuration.
- **Shared Endpoint**: This resource writes to the same `identitySynchronization` policy object as `microsoft365_graph_beta_identity_and_access_cross_tenant_access_partner_user_sync_settings`. Avoid managing both resources for the same partner tenant, as each performs a full replacement (PUT) of the policy and will overwrite the other's settings.

## Example Usage

```terraform
resource "microsoft365_graph_beta_identity_and_access_cross_tenant_access_partner_group_sync_settings" "example" {
  tenant_id    = "12345678-1234-1234-1234-123456789012"
  display_name = "Partner Organization Sync Configuration"

  group_sync_inbound = {
    is_sync_allowed = true
  }
}
```
