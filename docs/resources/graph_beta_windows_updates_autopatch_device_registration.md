# microsoft365_graph_beta_windows_updates_autopatch_device_registration (Resource)

Manages batch enrollment of Entra ID devices in Windows Autopatch update management using the `/admin/windows/updates/updatableAssets/enrollAssetsById` endpoint. This resource enrolls multiple devices for a specific update category (driver, feature, or quality updates). Devices are automatically created as azureADDevice resources in Windows Autopatch if they don't already exist. When the resource is deleted, devices are unenrolled from the specified update category.

## Dependency Notes

### Entra Device Object IDs

The `entra_device_object_ids` attribute requires **Entra ID device object IDs** (the `id` property from the Entra ID device object), **NOT** the `deviceId` property. 

- ✅ Correct: Use the object ID from `GET /devices/{id}` (e.g., `23ace577-ee29-416f-8566-11c948310bff`)
- ❌ Incorrect: Do not use the `deviceId` property (e.g., `06771871-1375-494e-97f9-ab87ba64edeb`)

You can obtain the correct object ID from:
- The `microsoft365_graph_beta_device_management_managed_device` data source using the `azure_ad_device_id` attribute
- The Entra ID portal under Devices → All Devices → Select device → Object ID
- Microsoft Graph API: `GET https://graph.microsoft.com/beta/devices`

### Update Categories

Each resource instance manages enrollment for a single update category:
- `quality` - Security and quality updates
- `feature` - Windows feature updates
- `driver` - Driver updates

To manage multiple update categories, create separate resource instances.
