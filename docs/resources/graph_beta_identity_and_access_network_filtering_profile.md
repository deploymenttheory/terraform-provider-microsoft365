# microsoft365_graph_beta_identity_and_access_network_filtering_profile (Resource)

Manages Microsoft Entra Global Secure Access security profiles, represented in Microsoft Graph beta as filtering profiles, using the `/networkAccess/filteringProfiles` endpoint.

This resource manages the Microsoft Entra admin center Global Secure Access **Security profiles** objects represented by Microsoft Graph beta as `microsoft.graph.networkaccess.filteringProfile`.

The resource manages the profile object itself. Filtering policy links returned from the profile's `policies` navigation property and Conditional Access policy references returned from `conditionalAccessPolicies` are not managed by this resource. Conditional Access Global Secure Access filtering profile assignments are managed by the Conditional Access policy resource using the `global_secure_access_filtering_profile.profile_id` setting.

## Microsoft Graph API

This resource uses the beta `/networkAccess/filteringProfiles` endpoint.

The Microsoft Graph beta metadata exposes the writable profile fields used by this resource: `name`, `description`, `state`, and `priority`. The Entra admin center Security profiles page was observed listing profiles with:

```http
GET /beta/networkaccess/filteringProfiles?$expand=policies($expand=policy),ConditionalAccessPolicies
```

Microsoft Learn currently documents list, get, and update operations for filtering profiles. The generated Microsoft Graph beta Go SDK exposes collection `Post` and item `Delete` request builders for `/networkAccess/filteringProfiles`; this resource uses those SDK builders for create and delete to stay aligned with the Graph service surface.

## Microsoft Graph API Permissions

The following client `application` permissions are needed in order to use this resource:

**Required:**
- `NetworkAccess.Read.All`
- `NetworkAccess.ReadWrite.All`

**Optional:**
- `None` `[N/A]`

## Default Profiles and Import

Entra tenants can contain a built-in baseline filtering profile. Portal traffic observed for this resource returned a `Baseline profile` at priority `65000`. Import existing profiles before managing them with Terraform. Avoid deleting Microsoft-created baseline profiles unless you have verified that the tenant allows removal and that the profile is not required by your Global Secure Access configuration.

## Example Usage

```terraform
resource "microsoft365_graph_beta_identity_and_access_network_filtering_profile" "enabled_profile" {
  name        = "Enabled Security Profile"
  description = "Global Secure Access security profile managed by Terraform"
  priority    = 100
  state       = "enabled"
}

resource "microsoft365_graph_beta_identity_and_access_network_filtering_profile" "disabled_profile" {
  name        = "Disabled Security Profile"
  description = "Global Secure Access security profile staged by Terraform"
  priority    = 200
  state       = "disabled"
}
```
