# microsoft365_graph_beta_identity_and_access_cross_tenant_access_partner_user_sync_settings (Resource)

Manages cross-tenant user synchronization settings for a specific partner tenant in Microsoft Entra ID using the `/policies/crossTenantAccessPolicy/partners/{id}/identitySynchronization` endpoint. This resource is used to configure whether users from a partner Microsoft Entra tenant can be synchronized to your tenant.

## Important Notes

- **Partner-Specific Configuration**: This resource manages user synchronization settings for a specific partner tenant. Each partner tenant requires its own resource instance.
- **Tenant ID Validation**: The `tenant_id` is validated against Microsoft Entra ID to ensure it represents a valid organization before creating the resource.
- **RequiresReplace**: Changing the `tenant_id` will force replacement of the resource, as it's used as the identifier.
- **User Sync Direction**: Currently, only inbound user synchronization (from partner to your tenant) is supported via the `user_sync_inbound` configuration.

## Example Usage

```terraform
resource "microsoft365_graph_beta_identity_and_access_cross_tenant_access_partner_user_sync_settings" "example" {
  tenant_id    = "12345678-1234-1234-1234-123456789012"
  display_name = "Partner Organization Sync Configuration"

  user_sync_inbound = {
    is_sync_allowed = true
  }
}
```
