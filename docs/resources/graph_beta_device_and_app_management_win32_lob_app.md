# microsoft365_graph_beta_device_and_app_management_win32_lob_app (Resource)

Manages Win32 lob applications using the `/deviceAppManagement/mobileApps` endpoint. This resource is used to win lob apps enable deployment of vendor supplied Windows applications (.msi, .appx, .appxbundle, .msix, and .msixbundle), and dependency management for enterprise software distribution. These installers do not need to be wrapped in the .intunewin file type.'https://learn.microsoft.com/en-us/intune/intune-service/apps/apps-win32-app-management'.

## Example Usage

### MSI-Based Win32 LOB App (7-Zip)

```terraform
resource "microsoft365_graph_beta_device_and_app_management_win32_lob_app" "seven_zip" {
  display_name    = "7-Zip 24.09 x64"
  description     = "7-Zip is a file archiver with a high compression ratio. Supports 7z, ZIP, CAB, RAR, ARJ, LZH, CHM, CPIO, CramFS, DEB, DMG, FAT, HFS, ISO, LZH, LZMA, MBR, MSI, NSIS, NTFS, RAR, RPM, SquashFS, UDF, VHD, WIM, XAR and Z formats."
  publisher       = "Igor Pavlov"
  product_code    = "{23170F69-40C1-2702-2409-000001000000}"
  product_version = "24.09.0.0"
  file_name       = "7z2409-x64.msi.intunewin"

  categories = [
    "Business",
    "Productivity",
  ]

  information_url = "https://www.7-zip.org/"
  developer       = "Igor Pavlov"
  owner           = "IT"
  notes           = "Standard MSI deployment. Silent install handled by Intune LOB app engine."

  app_installer = {
    installer_file_path_source = "/path/to/7z2409-x64.msi.intunewin"
  }

  app_icon = {
    icon_url_source = "https://upload.wikimedia.org/wikipedia/commons/thumb/2/21/7-zip-logo.svg/480px-7-zip-logo.svg.png"
  }

  command_line = "/qn REBOOT=ReallySuppress"
}
```

## Important Notes

- **Supported Formats**: Win32 LOB apps support `.msi`, `.appx`, `.appxbundle`, `.msix`, and `.msixbundle` installer formats. The installer file must still be wrapped in the `.intunewin` format before upload.
- **Simpler than Win32 App**: This resource does not support custom detection rules, requirement rules, return codes, or install experience configuration. Intune manages those aspects natively for LOB installers.
- **Product Code**: For MSI-based apps, providing the `product_code` (MSI GUID) enables Intune to detect and manage the installed application. Use `msiexec /qn` or `orca.exe` to retrieve the product code from the MSI.
- **Product Version**: Required. Used by Intune to track the installed version of the application.
- **Command Line**: Optional silent install arguments passed to the installer by Intune. For MSI packages, common flags are `/qn REBOOT=ReallySuppress`.
- **Content Upload**: The resource handles uploading the app content to Intune. The `.intunewin` file is encrypted and chunked automatically.
- **Assignment Required**: Apps must be assigned to user or device groups to be deployed. Use `microsoft365_graph_beta_device_and_app_management_mobile_app_assignment` for assignments.
